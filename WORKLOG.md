### 実機確認チェックリスト

テスト前に、取消・追加が発生してよいテスト会員と対象日を決める。実運用会員で不用意に試さない。

1. TASK-ATT-003-02 料金枠判定
   - [ ] スマートフォンで出席画面を開く。
   - [ ] 料金枠IDなしの道場QRを読み、現在時刻から期待する料金枠が自動判定される。
   - [ ] 画面の道場名、料金枠名、稽古日、選択可能な各枠の時刻がマスタと一致する。
   - [ ] 時刻上の候補が複数となる条件では、勝手に確定せず画面に手動選択候補を表示する。

2. TASK-ATT-003-03 登録済み枠復元
   - [ ] あらかじめテスト会員に対象日の出席枠を1件登録する。
   - [ ] 会員QRを読み、登録済み枠が選択状態で復元される。
   - [ ] 未登録会員では選択枠0件として表示される。
   - [ ] 状態取得中・取得失敗中は送信できない。

3. TASK-ATT-003-04 枠追加・解除
   - [ ] 復元された既存枠を1つ解除し、別の枠を1つ追加する。
   - [ ] 送信後、追加枠は有効行として1件追加され、解除枠は取消になる。
   - [ ] 変更しなかった枠は重複登録されない。
   - [ ] 再度同じ内容を送っても追加・取消が増えない。
   - [ ] 全枠を解除して送信すると、対象範囲の既存枠がすべて取消になる。

4. TASK-ATT-003-05 送信後再表示
   - [ ] 送信後の再取得結果が、画面で選択した枠と一致する。
   - [ ] 送信直後の再取得が成功し、「画面と出席ログの一致を確認」の成功ログが出る。
   - [ ] 同じ会員QRを再読取し、直前に送信した枠が復元される。
   - [ ] スプレッドシートの有効行が再表示の選択状態と一致する。
