# TASK-ATT-020 級段位・審査進捗表示

## Origin

- `docs/ETC/道場システム_ヒアリング対応項目_2026-07-21.md`
- 先生ヒアリング: 当日の参加者構成から稽古内容を決めたい。
- 現行出席管理表: 級段位がなく、先生の記憶・事前確認へ依存している。

## Scope

1. `01_会員マスタ`へ級段位・審査進捗の基礎情報を追加する。
2. 出席状態取得時に氏名・級段位・稽古数・審査可能数を返す。
3. 出席登録時に会員の自己申告級段位を反映する。
4. 既存出席画面へ級段位と進捗を表示する。
5. 会員向けサマリーAPIを追加する。
6. 先生画面へ本日の登録済み出席者一覧を追加する。

## Member master fields

- 現在級段位
- 級段位登録元
- 級段位更新日時
- 級段位起算日
- 繰越稽古数
- 審査可能稽古数

審査可能稽古数は正式基準確認前のためコードへ固定せず、当面は会員マスタに保持する。

## Acceptance

- 既存級段位が出席画面にデフォルト表示される。
- 級段位を変更して出席登録すると、会員マスタへ `本人申告` として反映される。
- 同日の複数稽古枠は、審査用稽古数では1日として数える。
- 級段位起算日以降の有効な出席日と繰越稽古数が合算される。
- 審査可能数が設定済みの場合、残り回数と到達状態を返す。
- 既存の出席登録・取消・課金枠計算を壊さない。
- 道場・課金枠の確定後、本日の登録済み出席者が自動表示される。
- 本日の出席者を氏名、級段位、審査の近さで並び替えられる。
- 先生、道場、課金枠は別々のQRから、読み取り順に依存せず設定できる。
- 道場と課金枠が揃った時点で組み合わせを照合し、本日の既存出席を表示する。
- `14_級段位マスタ` から成人・少年を別資格として級段位候補と表示順を取得する。
- 少年部は準十級から一級までの準級・本級、少年初段、少年二段を独立した級段位として扱う。
- `15_審査基準マスタ` の現在級段位から、次回審査候補と必要稽古数を取得する。
- 会員マスタの `審査可能稽古数` が設定されている場合は、個別上書き値として優先する。
- 三段以上など自動判定対象外の基準は、当面個別確認として扱う。
- 必要稽古数へ到達しても級段位を自動更新せず、先生確認の判断材料として表示する。
- 旧表記の級段位は成人・少年へ自動変換しない。

## Out of scope

- 審査基準マスタの正式化
- 先生による級段位確定
- 審査合否と級段位更新の連動
- 合気会共通管理表のPDF生成
