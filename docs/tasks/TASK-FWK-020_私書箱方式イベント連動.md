# 1. 目的

ARCH-013で定義した私書箱方式を実装し、出席事実から会費請求候補などの関連業務を疎結合に起動できるようにする。

---

# 2. 現在の対応範囲

- 出席登録成功後に `attendance_postEvent` を呼ぶ。
- 入口はイベントを保存・配送せず、実装保留であることを結果に返す。

ここまでを暫定対応とし、以降は保留する。

---

# 3. 再開後の実装候補

- イベント原簿
- イベント購読マスタ
- 購読先ごとの私書箱と処理状態
- イベントID、相関ID、重複防止キー
- 配送、再試行、エラー記録
- 汎用イベント実行Runner
- `billingCandidate_registerEventSubscriptions`
- `billingCandidate_handleEvent`
- 出席実績からの会費請求候補計算

---

# 4. 対象外

- 出席登録処理から会費処理を直接呼び出すこと
- イベント受信だけで請求を自動確定すること
- 人の確認を経ずに不可逆な業務処理を行うこと

---

# 5. 完了条件

- 発行元が購読先を知らずにイベントを投函できる。
- 購読先ごとに未処理、処理中、完了、失敗を管理できる。
- 同一イベントを再実行しても業務データが重複しない。
- 会費候補を先生が任意のタイミングで生成・確認できる。
- イベント処理失敗が出席登録の成功を取り消さない。
