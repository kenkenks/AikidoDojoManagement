# STORY-902 第3層 画面スモーク

## 目的

総合結合Runnerが`2099-07`へ残した結果を、実ブラウザ、実GAS、実QRで確認する。
内部データの再検証ではなく、利用者が日時、出席、会費受付、会費状況を理解できることを確認する。

## 前提

- `runner_story_integration_902()`が7/7 PASSしている。
- GASとGitHub Pagesへ最新コードを反映済みである。
- スプレッドシートを再読込し、`道場管理 > テスト時刻を設定`を表示できる。
- 先生、道場、課金枠は別々のQRとして用意する。

## 共通合格条件

- 各対象画面の上部にシステム日時と対象月が表示される。
- タイムトリップ中は黄色の`⚠ テスト時刻`表示になる。
- 画面の日付とGASが処理する日付が一致する。
- QRは元URLではなく、読み取ったパラメータを現在の画面コンテキストへ合成できる。
- エラー時に画面が無反応にならず、理由が表示される。

## Smoke 1: システム時刻

1. `道場管理 > テスト時刻を設定`を開く。
2. 有効、`2099-07-02 10:20:00`、対象月`2099-07`を保存する。
3. 先生ホームを再読込する。
4. `⚠ テスト時刻：2099-07-02 10:20:00 ／ 対象月：2099-07`を確認する。

1. `attendanceCheck.html?location_id=HONBU`を開く。
2. 稽古日が`2099-07-02`であることを確認する。
