# 1. 目的

通常稽古日における出席受付の運用を確認する。

本Storyは、通常稽古日に会員が自身の端末で出席登録を行い、先生が出席者を確認するまでの一連の運用を対象とする。

# 3. 前提条件

- 本日は通常稽古日である。
- 先生が道場に到着し、稽古の準備が完了している。
- 出席受付QRシート（道場）が受付場所に設置されている。
- 会員は出席登録画面を利用できる。
- 会員は会員カードまたは月謝袋のQRを所持している。
- 会費は未払いである。

---

# 4. シナリオ

## Step01

**先生**

出席受付QRシート（道場）を受付場所へ設置する。

---

## Step02

**会員**

出席登録画面を開く。

---

## Step03

**会員**

出席受付QRシート（道場）を読み取る。

---

## Step04

**会員**

会員カードまたは月謝袋のQRを読み取り、出席登録を行う。

---

## Step05

**会員**

出席登録が完了したことを確認する。

---

## Step06

**先生**

当日の出席者一覧を確認する。

一覧には、会員が自身で登録した出席情報が「確認待ち」として表示される。

先生は出席内容を確認し、確認ボタンを押す。

**期待結果**

- 会員の出席情報が「確認待ち」として表示される。
- 先生が確認すると、出席情報が「確認済み」となる。
- 確認済みの出席情報には、確認した先生が登録される。

---

# 5. フロー

```
出席受付QRシート（道場）を設置
            ↓
会員がセルフ出席登録
            ↓
teacher_id = NULL
status = 確認待ち
            ↓
先生が出席者一覧を確認
            ↓
先生が確認ボタンを押す
            ↓
teacher_id を登録
status = 確認済
```

---

# 6. 確認事項

- [ ] 出席受付QRシートを設置するだけで受付を開始できること。
- [ ] 会員が自身の端末だけで出席登録を完了できること。
- [ ] 出席登録が正常に完了すること。
- [ ] 同じQRを連続して読み取った場合の動作を確認すること。
- [ ] 先生が当日の出席者一覧を確認できること。
- [ ] 先生が確認すると、出席状態が「確認済み」となること。
- [ ] 確認した先生が出席情報に記録されること。
