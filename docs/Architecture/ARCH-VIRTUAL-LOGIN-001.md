# ARCH-VIRTUAL-LOGIN-001 仮想ログイン

## 目的

先生・会員QRから取得した操作主体を、同一タブ内の画面遷移中に保持する。

## Reality

- 保存先はブラウザの `sessionStorage` とする。
- 先生は `TEACHER + teacher_id`、会員は `MEMBER + member_id` を保持する。
- タブを閉じた場合、または明示ログアウト時に状態を破棄する。
- タイムアウト判定は今回実装しない。ただし将来追加用に `logged_in_at` と `last_accessed_at` を保持する。
- 入口で人物QRを読んだ場合だけログイン主体を切り替える。
- 先生画面で読み取る会員QRは業務対象であり、先生のログイン主体を会員へ切り替えない。
- 道場・課金枠は業務コンテキストであり、ログイン主体には含めない。

## セキュリティ上の位置付け

これは画面操作を便利にする仮想ログインであり、本人認証・認可ではない。QRの所持確認、署名、サーバーセッション等は将来の本認証で扱う。
